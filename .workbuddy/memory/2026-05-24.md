## Fix: Auth logic not enforced (basic-multi-user)

**Root cause**: In `core/match.go`, the three locations that construct `service.Service{}` from the rule config
(`hostMatcherFromMatchedRule` — exact branch, `hostMatcherFromMatchedRule` — rewriter/regex branch,
`pathMatchResultWithHost`, and the fallback HostMatcher block) only copied `Protocol`, `Name`, `Port`,
`Request`, `Response`. The `Auth` field (and `Mode`, `StripPrefix`, `HealthCheck`) were silently dropped.

Because `ValidateAuth` reads from the returned `Service.Auth`, it always saw an empty struct → auth type ""
→ unconditional pass. All requests bypassed authentication.

**Fix**: Added `Auth`, `Mode`, `StripPrefix`, `HealthCheck` to all four `service.Service{}` literal
constructions in `core/match.go`.

**Tests added**:
- `core/match_test.go`: 3 regression unit tests verifying `MatchHost` returns correct `Auth` field for
  exact host, bearer token host, and regex host patterns.
- `core/build_test.go`: 8 HTTP integration tests (using `httptest.NewRecorder`) covering:
  - BasicAuth: no creds → 401 + WWW-Authenticate header
  - BasicAuth: wrong creds → 401
  - BasicAuth multi-user: all 3 valid users pass (200)
  - BasicAuth multi-user: wrong creds → 401
  - BearerAuth: no creds → 401 + WWW-Authenticate header
  - BearerAuth: wrong token → 401
  - No-auth route: still returns 200 (regression guard)

All `go test ./core/...` pass.

## Fix: Overview metrics always shows 0 despite access log data (3da7c80)

**Root cause**: zoox file transport writes ANSI-colored log levels (e.g. `\x1b[34mINFO\x1b[39m`) and prepends
duplicate timestamps before access log lines. `parseAccessLine` in `core/admin/service/accesslog.go` had no
logic to handle these prefixes — the ANSI codes prevented the log level regex from matching, so all lines
were silently rejected as non-access-log entries.

**Fix**:
- Added `reANSI` to strip ANSI escape sequences before parsing
- Added `reLogTime2` and `reLogLev` to strip the second timestamp and log level prefix from zoox file transport format
- Added `access_log_parse_fail` source diagnostic in metrics.go
- Added test cases for ANSI-colored and zoox-prefixed log lines
- Also included: OverviewPage auto-refresh polling (5s), improved empty state messages
